# 브랜드 포털에서 폴더 공유 {#share-folders}

브랜드 포털은 자산 수집을 지원하지 않으므로, 사전 구성된 AEM 작성자 인스턴스에서 브랜드 포털에 자산을 게시해야 합니다.

## 브랜드 포털의 폴더 공유 워크플로 {#folder-sharing-workflow-in-brand-portal}

다음은 폴더 공유 워크플로 및 사용자 액세스에 대해 설명합니다.

* 기본적으로 AEM 자산에서 브랜드 포털에 게시된 모든 폴더는 복제를 구성하는 동안 공개로 표시되지 않는 한 브랜드 포털 관리자만 볼 수 있습니다.
* 관리자는 폴더 [!UICONTROL 속성] 콘솔을 사용하여 선택적 사용자 또는 그룹과 폴더를 공유합니다. 폴더를 공유한 사용자나 그룹만 브랜드 포털에 로그인한 후에 폴더를 볼 수 있습니다. 다른 사용자는 폴더를 볼 수 없습니다.
* 관리자는 [폴더 속성] 콘솔의 [공용 폴더] [!UICONTROL 확인란을] 통해 폴더를 공개하도록 선택할 [!UICONTROL 수] 있습니다. 공용 폴더는 모든 사용자가 볼 수 있습니다.

* 사용자 역할 및 권한에 관계없이 사용자가 브랜드 포털에 로그인하면 모든 공용 폴더 및 사용자와 직접 공유된 폴더 또는 사용자가 속한 그룹과 폴더가 표시됩니다. 비공개 폴더 또는 다른 사용자와 공유된 폴더는 모든 사용자에게 표시되지 않습니다.

### 브랜드 포털에서 사용자 그룹과 폴더 공유 {#sharing-folders-with-user-groups-on-brand-portal}

폴더의 자산에 대한 액세스 권한은 하위 폴더의 설정에 관계없이 상위 폴더의 액세스 권한에 따라 달라집니다. 하위 폴더가 상위 폴더에서 [ACL을 상속하므로](https://helpx.adobe.com/experience-manager/6-5/sites/administering/using/security.html#PermissionsinAEM) 이 동작은 AEM의 ACL을 따릅니다. 예를 들어, 폴더 A에 폴더 C가 포함된 폴더 B가 포함되어 있는 경우 폴더 A에 대한 액세스 권한이 있는 사용자 그룹(또는 사용자)도 폴더 B와 폴더 C에 동일한 액세스 권한을 갖습니다. 폴더 B는 해당 ACL을 상속하고 폴더 C는 B의 하위 폴더가 되면 해당 ACL을 상속합니다.

마찬가지로, 폴더 B만 액세스할 수 있는 권한이 있는 사용자 그룹(또는 사용자)은 폴더 A에 대해 동일한 액세스 권한을 가지지만 폴더 C에 대해서는 액세스 권한이 없습니다.따라서 가장 많이 노출된 에셋이 하위 폴더에 배치되고 하위 폴더에서 루트 폴더 액세스까지 제한될 수 있도록 조직이 컨텐츠를 정렬하는 것이 좋습니다.

### Public folder publish {#public-folder-publish}

브랜드 [!UICONTROL 포털 복제를 구성하는 동안] [공용 폴더 게시] 옵션을 선택하지 않는 한, 관리자가 아닌 사용자(예: 편집기 및 뷰어)는 AEM 자산에서 브랜드 포털에 게시된 자산에 액세스할 수 없습니다.

![](assets/assetbpreplication.png)

[ [!UICONTROL 공용 폴더 게시] ] 옵션이 비활성화된 경우 관리자는 공유 기능을 사용하여 관리자가 아닌 사용자와 이러한 자산을 공유해야 합니다.

>[!NOTE]
>
>공용 폴더 게시를 [!UICONTROL 활성화하는 옵션은] AEM 6.3.2.1 이상 버전에서 사용할 수 있습니다.

## 공유 폴더에 대한 액세스 {#access-to-shared-folders}

다음 매트릭스에서는 다양한 사용자 역할에 대해 자산을 공유/공유 해제할 수 있는 액세스 권한 및 권한을 설명합니다.

|  | AEM 자산에서 브랜드 포털에 게시된 모든 폴더에 대한 액세스 | 공유 폴더에 대한 액세스 | 폴더 권한 공유/공유 취소 |
|---------------|-----------|-----------|------------|
| 관리자 | 예 | 예 | 예 |
| 편집자 | 아니오* | 예, 해당 사용자와 공유하거나 해당 그룹이 속한 그룹과 공유하는 경우에만 가능합니다. | 예, 공유된 폴더 또는 해당 폴더가 속한 그룹에 대해서만 |
| 뷰어 | 아니오* | 예, 해당 사용자와 공유하거나 해당 그룹이 속한 그룹과 공유하는 경우에만 가능합니다. | 아니오 |
| 게스트 사용자 | 아니오* | 예, 해당 사용자와 공유하거나 해당 그룹이 속한 그룹과 공유하는 경우에만 가능합니다. | 아니오 |

**기본적으로 AEM 작성자와[!UICONTROL 함께 브랜드]포털의 복제를 구성하는 동안 공용 폴더 게시 옵션이 비활성화됩니다. 이 옵션을 활성화하면 기본적으로 모든 사용자(관리자가 아닌 사용자도)가 브랜드 포털에 게시된 폴더에 액세스할 수 있습니다.*

### 관리자가 아닌 사용자가 공유 폴더에 액세스 {#non-admin-user-access-to-shared-folders}

관리자가 아닌 사용자는 브랜드 포털에서 공유된 폴더만 액세스할 수 있습니다. 그러나 이러한 폴더가 로그인할 때 포털에서 표시되는 방식은 폴더 계층 구조 활성화 [!UICONTROL 구성의 설정에 따라 달라집니다] .

**구성이 비활성화된 경우**

관리자가 아닌 사용자는 브랜드 포털에 로그인하면 랜딩 페이지에서 자신과 공유된 모든 폴더를 볼 수 있습니다.

![](assets/disabled-folder-hierarchy1-1.png)

**구성이 활성화된 경우**

관리자가 아닌 사용자는 브랜드 포털에 로그인할 때 폴더 트리(루트 폴더에서 시작)와 해당 상위 폴더 내에 배열된 공유 폴더를 볼 수 있습니다.

이러한 상위 폴더는 가상 폴더이며, 가상 폴더에 대해 작업을 수행할 수 없습니다. 이러한 가상 폴더는 잠금 아이콘을 사용하여 인식할 수 있습니다.

공유 폴더와 달리 카드 보기에서 마우스로 가리키거나 선택할 [!UICONTROL 때]작업 작업을 볼 수 없습니다. [!UICONTROL [개요] ] 단추는 [열 보기] 및 [목록 보기]에서 가상 [!UICONTROL 폴더를] 선택할 때 [!UICONTROL 표시됩니다].

>[!NOTE]
>
>가상 폴더의 기본 축소판은 첫 번째 공유 폴더의 축소판 이미지입니다.

![](assets/enabled-hierarchy1-1.png) ![](assets/hierarchy1-nonadmin-1.png) ![](assets/hierarchy-nonadmin-1.png) ![](assets/hierarchy2-nonadmin-1.png)

## 폴더 공유 {#how-to-share-folders}

브랜드 포털에서 사용자와 폴더를 공유하려면 다음 단계를 따르십시오.

1. 왼쪽의 오버레이 아이콘을 클릭하고 [탐색]을 **[!UICONTROL 선택합니다]**.

   ![](assets/selectorrail.png)

2. 왼쪽에 있는 사이드레일에서 파일을 **[!UICONTROL 선택합니다]**.

   ![](assets/access_files.png)

3. 브랜드 포털 인터페이스에서 공유할 폴더를 선택합니다.

   ![](assets/share-folders.png)

4. 맨 위의 도구 모음에서 공유를 **[!UICONTROL 선택합니다]**.

   ![](assets/share_icon.png)

   폴더 [!UICONTROL 속성] 콘솔이 나타납니다.

   ![](assets/folder_properties.png)

5. 기본 [!UICONTROL 이름을 사용자에게 표시하지 않으려면] 폴더 속성 [!UICONTROL 콘솔에서 폴더] 제목필드에 폴더 제목을 지정합니다.
6. 사용자 [!UICONTROL 추가] 목록에서 폴더를 공유할 사용자나 그룹을 선택하고 추가를 **[!UICONTROL 클릭합니다]**.
손님 사용자만 이용하고 다른 사용자는 공유하지 않으려면 [구성원] **[!UICONTROL 드롭다운에서]** [익명 사용자]를 [!UICONTROL 선택합니다] .

   ![](assets/only-anonymous.png)

   >[!NOTE]
   >
   >그룹 구성원 및 역할에 관계없이 모든 사용자가 폴더를 사용할 수 있도록 하려면 [공용 폴더] **[!UICONTROL 확인란을 선택하여]** 공개하십시오.

7. 필요한 경우 축소판 **[!UICONTROL 변경을]** 클릭하여 폴더의 축소판 이미지를 수정합니다.
8. **[!UICONTROL 저장을 클릭합니다]**.
9. 공유 폴더에 액세스하려면 폴더를 공유한 사용자의 자격 증명으로 브랜드 포털에 로그인합니다. 인터페이스에서 공유 폴더를 검토합니다.

## 폴더 공유 취소 {#unshare-the-folders}

이전에 공유한 폴더의 공유를 취소하려면 다음 단계를 따르십시오.

1. 브랜드 포털 인터페이스에서 공유를 해제할 폴더를 선택합니다.

   ![](assets/share-folders-1.png)

2. 맨 위의 도구 모음에서 공유를 **[!UICONTROL 클릭합니다]**.
3. 폴더 [!UICONTROL 속성] 콘솔의 [!UICONTROL 멤버]에서 **[!UICONTROL 사용자 옆에 있는]** x기호를 클릭하여 폴더를 공유한 사용자 목록에서 제거합니다.

   ![](assets/folder_propertiesunshare.png)

4. 경고 메시지 상자에서 확인을 클릭하여 **[!UICONTROL 공유]** 해제를 확인합니다.
**[!UICONTROL 저장을 클릭합니다]**.

5. 공유 목록에서 제거한 사용자의 자격 증명으로 브랜드 포털에 로그인합니다. 사용자의 브랜드 포털 인터페이스에서 더 이상 폴더를 사용할 수 없습니다.
